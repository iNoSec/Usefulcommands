If someone else runs the file (ie not root), they will run the file as the user/group who created it (ie look for files created as root. when exec by non root, they will still run as root).

Using it on a directory is slightly different. "Executing" a directory means the possiblity to access anythign in the dirctory (either file or subdir). This is different from read (r) permission on a directory, which allows you ro tread the names of the files/subdirectories.

Using chmod +s on a directory, changes the user/group as which you "execute" the directory. This implie that, whenver a new file or subdir is created, it will "inherit" the group ownership of the parent directory if the "setGID" bit is set. Inheriting the owner is not possible. Ownership of new files/subdirs always belongs to the user who created it.

Using the find command to locate suid and guid files is a common practice to locate candidate programs to uninstall
IE: you can use the find command to help locate some files that may run with different permissions
